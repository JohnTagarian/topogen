## 1. ภาพรวมของระบบ

**AI Network Topology Generator** คือเครื่องมือที่ใช้ AI ออกแบบโครงสร้างเครือข่าย (Network Topology) และจัดทำรายการอุปกรณ์พร้อมราคา (Bill of Materials) โดยอัตโนมัติ เพียงแค่อธิบายความต้องการเป็นภาษาธรรมชาติ ทั้งภาษาไทยและภาษาอังกฤษ

### ระบบทำอะไรให้คุณได้บ้าง

| ผลลัพธ์ | รายละเอียด |
|---|---|
| **Network Diagram** | แผนภาพโครงสร้างเครือข่ายแบบ Interactive ลากโหนดได้ ซูมได้ |
| **Design Summary** | คำอธิบายแนวทางการออกแบบ พร้อมตาราง Subnet/VLAN |
| **Bill of Materials** | รายการอุปกรณ์ยี่ห้อจริง พร้อมราคาเป็นบาท (รวม VAT) |
| **Export PNG** | บันทึก Topology Diagram เป็นไฟล์รูปภาพความละเอียดสูง |
| **Export CSV** | บันทึก BOM เป็นไฟล์ Excel สำหรับทำงานต่อ |

### 3.2 ปุ่มตัวอย่าง (Example Pills)

แถบปุ่มด้านบน textarea มีตัวอย่าง 4 แบบ:

| ปุ่ม | ข้อความที่จะถูกใส่ |
|---|---|
| ออฟฟิศ 3 ชั้น | ออฟฟิศ 3 ชั้น ชั้นละ 30 คน มี Server Room ต้องการ WiFi ทุกชั้น |
| โรงเรียน 200 คน | โรงเรียน 200 คน ต้องการ internet และ file server |
| โรงงาน 3 อาคาร | โรงงาน 3 อาคาร พนักงาน 100 คน ต้องการ VoIP และ CCTV |
| Small office | Small office 10 users, need internet and WiFi, budget 50000 THB |

> **วิธีใช้:** คลิกที่ปุ่มตัวอย่าง ข้อความจะถูกใส่ใน textarea อัตโนมัติ แก้ไขได้ตามต้องการก่อนกด Generate

### 3.3 การกรอก Requirement

พิมพ์ความต้องการเครือข่ายใน textarea ระบุข้อมูลให้ได้มากที่สุด:

| ข้อมูล | ตัวอย่าง |
|---|---|
| ประเภทองค์กร | ออฟฟิศ, โรงเรียน, โรงงาน, โรงพยาบาล |
| จำนวนผู้ใช้ | 50 คน, 200 users |
| จำนวนชั้น / อาคาร | 3 ชั้น, 2 อาคาร |
| บริการที่ต้องการ | WiFi, VoIP, CCTV, File Server, Internet |
| งบประมาณ | งบประมาณ 200,000 บาท |
| ความปลอดภัย | ต้องการ Firewall, ระดับความปลอดภัยสูง |

**ตัวอย่าง Requirement ที่ดี:**
```
ออฟฟิศ 4 ชั้น พนักงาน 120 คน แต่ละชั้น 30 คน
มี Server Room 1 ห้องที่ชั้น 1 ต้องการ WiFi ทุกชั้น
VoIP สำหรับโทรศัพท์ภายใน ต้องการ Firewall
งบประมาณประมาณ 500,000 บาท
```

### 3.4 ปุ่ม Clear (X)

- ปุ่ม **X** จะปรากฎที่มุมบนขวาของ textarea เมื่อมีข้อความอยู่
- คลิกเพื่อล้างข้อความทั้งหมดและเริ่มพิมพ์ใหม่

### 3.5 ปุ่ม Generate Topology

เมื่อกรอกข้อมูลครบแล้ว คลิกปุ่ม **✨ Generate Topology**

ระหว่างประมวลผล ปุ่มจะเปลี่ยนเป็น:
```
⏳ Generating Topology… Please wait
```

> ระบบใช้เวลาประมวลผลประมาณ **10–30 วินาที** ขึ้นอยู่กับความซับซ้อนของ requirement  
> กรุณารอจนระบบตอบกลับ **อย่ากดปุ่มซ้ำ** เพราะปุ่มถูก disable ไว้อัตโนมัติ

### 4.2 Network Topology Diagram

แผนภาพโครงสร้างเครือข่ายแบบ Interactive แสดงอุปกรณ์ทุกชิ้นและเส้นเชื่อมต่อ

#### ไอคอนอุปกรณ์

| ลักษณะไอคอน | อุปกรณ์ | สี |
|---|---|---|
| วงกลม + เส้นไขว้ | Router | น้ำเงินเข้ม |
| สี่เหลี่ยมแนวนอนมีพอร์ต | Switch | เขียว |
| สี่เหลี่ยมมีช่องตาราง | Firewall | แดง |
| กล่องตั้งมีไฟ LED | Server | เทาเข้ม |
| เส้นโค้งสัญญาณ WiFi | Access Point | น้ำเงิน |
| จอคอมพิวเตอร์ | PC / Endpoint | เทา |
| รูปเมฆ | Cloud / ISP | เทากลาง |
| สายฟ้าในกล่องสีส้ม | UPS | ส้ม |

#### การจัด Layout (Hierarchical)

โหนดถูกจัดเรียงจากบนลงล่างตามลำดับชั้นเครือข่าย:

```
          [Cloud / ISP]
                │
           [Firewall]              ← Core Layer
                │
            [Router]
                │
          [Core Switch]            ← Distribution Layer
           /          \
    [Switch F1]   [Switch F2]      ← Access Layer
     /     \        /     \
 [AP]  [Server]  [AP]   [PC]      ← Endpoint Layer
```

#### รูปแบบเส้นเชื่อมต่อ

| ลักษณะเส้น | ความหมาย |
|---|---|
| เส้นทึบ สีเทาเข้ม | Ethernet / Fiber |
| เส้นประ สีฟ้า | Wireless / VPN |

#### การควบคุม Diagram

| การกระทำ | ผลลัพธ์ |
|---|---|
| คลิก **⊞ Fit** | รีเซ็ต zoom ให้เห็น topology ทั้งหมด |
| คลิก **+ Zoom In** | ซูมเข้า 1.3 เท่า |
| คลิก **− Zoom Out** | ซูมออก |
| เลื่อน Scroll wheel | ซูมเข้า / ออก |
| คลิกค้างแล้วลาก | เลื่อนมุมมอง (Pan) |
| คลิก **💾 Save PNG** | บันทึก Diagram เป็นไฟล์ภาพ |

### 4.3 Design Summary

**Design Notes**  
AI จะอธิบาย 2–3 ประโยคว่าทำไมถึงเลือก Topology แบบนั้น เช่น เหตุผลด้านจำนวนผู้ใช้, การแยก VLAN, หรือการรองรับ redundancy

**ตาราง Subnet / VLAN**

| คอลัมน์ | ตัวอย่าง | ความหมาย |
|---|---|---|
| Subnet | LAN-Floor1 | ชื่อ Network segment |
| CIDR | `192.168.1.0/24` | ช่วง IP Address |
| VLAN ID | VLAN 10 | หมายเลข VLAN (แสดงเป็น badge สีเขียว) |

### 4.4 Bill of Materials (BOM)

รายการอุปกรณ์ที่แนะนำพร้อมราคาตลาดในประเทศไทย

**คอลัมน์ในตาราง:**

| คอลัมน์ | ความหมาย |
|---|---|
| Category | ประเภทอุปกรณ์ (Router, Switch, Firewall, …) |
| Brand / Model | ยี่ห้อและรุ่นที่แนะนำ |
| Specs | สเปคสำคัญโดยย่อ |
| Qty | จำนวนที่ต้องการ |
| Unit Price (฿) | ราคาต่อชิ้น |
| Subtotal (฿) | ราคารวมของอุปกรณ์นั้น |

**Summary ด้านล่างตาราง:**

| รายการ | ความหมาย |
|---|---|
| รวมทั้งหมด (ก่อน VAT) | ราคารวมก่อนภาษี |
| VAT 7% | ภาษีมูลค่าเพิ่ม |
| **รวมทั้งหมด (รวม VAT)** | **ราคาสุดท้าย** (ไฮไลท์สีฟ้า) |

> **หมายเหตุ:** ราคาเป็นข้อมูลอ้างอิงตลาด ณ เวลาพัฒนาระบบ ควรขอใบเสนอราคาจริงจากผู้จำหน่ายก่อนตัดสินใจ

## 5. การ Export ข้อมูล

### 5.1 บันทึก Topology เป็น PNG

1. ในส่วน Network Topology Diagram คลิกปุ่ม **💾 Save PNG**
2. ไฟล์ `network-topology.png` จะถูก download ทันที
3. ภาพมีความละเอียด **2x** พื้นหลังสีขาว เหมาะนำไปประกอบ proposal หรือเอกสาร

> **เคล็ดลับ:** กด **⊞ Fit** ก่อน Save PNG เพื่อให้ภาพแสดง topology ครบทั้งหมด

### 5.2 บันทึก BOM เป็น CSV

1. ในส่วน Bill of Materials คลิกปุ่ม **💾 Save CSV**
2. ไฟล์ `bom.csv` จะถูก download ทันที
3. เปิดได้ด้วย Microsoft Excel, Google Sheets, LibreOffice Calc

**รูปแบบไฟล์ CSV:**
```
Category,Brand,Model,Specs,Qty,Unit Price (THB),Subtotal (THB)
Firewall,Fortinet,FortiGate 60F,"10x GE, 700Mbps",1,35000,35000
Switch (PoE),Cisco,CBS250-24P,"24x GE PoE+, 4x SFP",3,18000,54000
...
,,,,,Total (before VAT),300000
,,,,,VAT 7%,21000
,,,,,Total (inc. VAT),321000
```

> **หมายเหตุ Excel:** หากภาษาไทยแสดงผิดใน Excel ให้ทำดังนี้  
> Excel → Data → From Text/CSV → เลือกไฟล์ → ตั้ง Encoding เป็น **UTF-8** → Load

## 6. ตัวอย่างการใช้งาน

### ตัวอย่างที่ 1: ออฟฟิศขนาดกลาง

**Input:**
```
บริษัทขนาดกลาง พนักงาน 80 คน ออฟฟิศ 2 ชั้น ชั้นละ 40 คน
มี Meeting Room 4 ห้อง ต้องการ WiFi ทั่วอาคาร
ใช้โทรศัพท์ IP (VoIP) ทุกโต๊ะ ต้องการ File Server กลาง
ระดับความปลอดภัยปานกลาง งบประมาณ 400,000 บาท
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology แบบ Tree / Hierarchical
- มี Firewall, Core Switch, Distribution Switch ต่อชั้น
- PoE Switch สำหรับ VoIP phones และ Access Point
- VLAN แยก Data / VoIP / Management

---

### ตัวอย่างที่ 2: โรงงานขนาดเล็ก

**Input:**
```
โรงงานผลิตชิ้นส่วน 2 อาคาร พนักงาน 60 คน
ต้องการ CCTV ทั่วโรงงาน WiFi ในส่วนออฟฟิศ
Internet ความเร็ว 100Mbps งบประมาณ 250,000 บาท
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology แบบ Star หรือ Hybrid
- PoE Switch สำหรับกล้อง CCTV
- Access Point ในส่วนออฟฟิศ
- Firewall สำหรับความปลอดภัย

---

### ตัวอย่างที่ 3: Small Office (ภาษาอังกฤษ)

**Input:**
```
Small startup office, 15 users, 1 floor
Need internet, WiFi, and a NAS for file sharing
Budget: 80,000 THB, basic security is fine
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology แบบ Star (เรียบง่าย ไม่ over-design)
- SMB Router + PoE Switch + Access Point + NAS
- ราคารวมอยู่ในงบ 80,000 บาท

## 7. คำถามที่พบบ่อย

**Q: ระบบรองรับภาษาอะไรบ้าง?**  
A: รองรับทั้งภาษาไทยและภาษาอังกฤษ สามารถผสมกันได้ในข้อความเดียวกัน

**Q: ใช้เวลานานแค่ไหนในการ Generate?**  
A: โดยปกติ 10–30 วินาที ขึ้นอยู่กับความซับซ้อนของ requirement และความเร็วอินเทอร์เน็ต

**Q: ราคาอุปกรณ์ใน BOM เชื่อถือได้แค่ไหน?**  
A: ราคาเป็นข้อมูลอ้างอิงเพื่อประมาณงบประมาณเบื้องต้นเท่านั้น ควรขอใบเสนอราคาจากผู้จำหน่ายก่อนตัดสินใจซื้อ

**Q: กลับไปแก้ Requirement แล้ว Generate ใหม่ได้ไหม?**  
A: ได้ คลิกปุ่ม **← ออกแบบใหม่** ที่ด้านบนขวาของหน้าผลลัพธ์

**Q: ทำไม Diagram บางครั้งโหนดเยอะมาก?**  
A: ระบบออกแบบตาม requirement ที่กรอก หากต้องการ topology เรียบง่ายกว่า ให้ระบุจำนวนผู้ใช้น้อยลง หรือเพิ่มคำว่า "minimal design" หรือ "ไม่ต้องซับซ้อน" ใน requirement

**Q: ไฟล์ CSV เปิดใน Excel แล้วตัวอักษรภาษาไทยเสีย?**  
A: ดู [หมายเหตุ Excel](#52-บันทึก-bom-เป็น-csv) ด้านบน หรือเปิดผ่าน Google Sheets แทน (รองรับ UTF-8 โดยอัตโนมัติ)

**Q: สามารถ Save ผลลัพธ์ไว้ดูทีหลังได้ไหม?**  
A: ปัจจุบันระบบยังไม่มีการบันทึกประวัติ แนะนำให้ Export PNG และ CSV ทันทีที่ได้ผลลัพธ์
