## 1. ภาพรวมระบบ

AI Network Topology Generator คือเครื่องมือที่ช่วยออกแบบโครงสร้างเครือข่ายคอมพิวเตอร์ (Network Topology) โดยอัตโนมัติ ผู้ใช้เพียงอธิบายความต้องการด้วยภาษาธรรมชาติ (ภาษาไทยหรืออังกฤษ) ระบบจะวิเคราะห์และส่งออกผลลัพธ์ 3 ส่วน คือ

1. **แผนภาพ Network Topology** — แสดงอุปกรณ์และการเชื่อมต่อทั้งหมด
2. **Bill of Materials (BOM)** — รายการอุปกรณ์ที่ต้องจัดซื้อพร้อมราคาอ้างอิง
3. **Design Summary** — คำอธิบาย logic การออกแบบ และตาราง Subnet/VLAN

ระบบเหมาะสำหรับ:
- นักศึกษาด้าน Network / IT
- วิศวกรเครือข่ายที่ต้องการ Draft ออกแบบเบื้องต้น
- ผู้จัดการโครงการที่ต้องการประมาณงบประมาณ
- ฝ่าย IT ขององค์กรขนาดเล็กถึงกลาง

## 5. วิธีเขียน Requirement ที่ดี

### 5.1 ข้อมูลที่ควรระบุ

| ข้อมูล | ตัวอย่าง |
|--------|---------|
| **ประเภทองค์กร** | ออฟฟิศ, โรงเรียน, โรงงาน, โรงพยาบาล, บ้าน |
| **จำนวนผู้ใช้** | 50 คน, 200 คน |
| **จำนวนชั้น/อาคาร** | 3 ชั้น, 2 อาคาร |
| **Service ที่ต้องการ** | WiFi, VoIP, CCTV, file server, internet |
| **ความต้องการด้าน Security** | ปกติ, ต้องการ Firewall, high security |
| **Redundancy** | ต้องการ Redundant link, HA |
| **งบประมาณ** | งบไม่เกิน 200,000 บาท |

### 5.2 ตัวอย่าง Requirement ที่ดี

**ตัวอย่างที่ 1 — ออฟฟิศขนาดกลาง (ภาษาไทย):**
```
ออฟฟิศ 3 ชั้น ชั้นละ 30 คน มี Server Room 1 ห้อง 
ต้องการ WiFi ทุกชั้น มี Firewall และ internet 1Gbps
งบประมาณไม่เกิน 500,000 บาท
```

**ตัวอย่างที่ 2 — โรงเรียน (ภาษาไทย):**
```
โรงเรียนมัธยม 2 อาคาร นักเรียน 800 คน ครู 60 คน
ต้องการ internet ทุกห้องเรียน WiFi ในอาคาร ห้อง Server 1 ห้อง
มีห้องปฏิบัติการคอมพิวเตอร์ 3 ห้อง ห้องละ 40 เครื่อง
```

**ตัวอย่างที่ 3 — โรงงาน (ภาษาไทย):**
```
โรงงานผลิต 3 อาคาร พนักงาน 150 คน ต้องการ VoIP, CCTV
มี server สำหรับระบบ MES ต้องการ redundancy สูง ไม่ให้ระบบล่ม
```

**ตัวอย่างที่ 4 — ภาษาอังกฤษ:**
```
Small dental clinic, 2 floors, 15 staff, need WiFi, internet,
file server for patient records, moderate security, budget 150000 THB
```

### 5.3 เทคนิคเพิ่มเติม

- **ยิ่งละเอียดยิ่งดี** — ระบุ service ที่ต้องการให้ครบ เพราะส่งผลต่อจำนวนอุปกรณ์และราคา
- **ระบุ budget** — ถ้าระบุ AI จะพิจารณา tier ของอุปกรณ์ให้เหมาะสม
- **ถ้าไม่แน่ใจ ไม่ต้องระบุ** — ระบบจะใช้ค่า default ที่เหมาะสม

## 6. หน้าผลลัพธ์

เมื่อ AI ประมวลผลเสร็จ จะแสดงผลลัพธ์ 4 ส่วน:

### 6.1 Network Topology Diagram

แผนภาพแสดงอุปกรณ์ทั้งหมดและการเชื่อมต่อ โดยจัดเรียงตาม Layer:

```
Layer Core (บนสุด):     Cloud/ISP → Firewall → Router
Layer Distribution:      Core Switch, File Server
Layer Access:            Floor Switches, Access Points
Layer Endpoint (ล่างสุด): User PCs, Printers
```

**สีของ Node แต่ละประเภท:**

| สี | ประเภทอุปกรณ์ |
|----|---------------|
| 🔵 ฟ้าอ่อน (Light Blue) | Cloud / ISP |
| 🔴 แดง (Red) | Firewall |
| 🟢 เขียวฟ้า (Teal) | Router |
| 🔵 น้ำเงิน (Blue) | Switch |
| 🟢 เขียวอ่อน (Mint) | Server |
| 🟡 เหลือง (Yellow) | Access Point (WiFi) |
| 🟣 ม่วง (Purple) | User PC |
| ⬜ เทา (Gray) | Printer |

**สีของ Link:**

| สี | ประเภทการเชื่อมต่อ |
|----|-------------------|
| แดง เส้นทึบ | Fiber Optic |
| ดำ เส้นทึบ | Ethernet |
| เทา เส้นประ | Wireless |

**ตัวเลขบน Link** = Bandwidth ของการเชื่อมต่อนั้น เช่น 1Gbps, 10Gbps

### 6.2 Design Summary

แสดง:
- **Topology Type** — ประเภทโครงสร้าง (Star / Tree / Mesh / Hybrid)
- **Design Notes** — คำอธิบาย Logic การออกแบบจาก AI
- **Subnet Table** — ตาราง IP Subnet และ VLAN ID ที่ออกแบบมาให้

ตัวอย่าง Subnet Table:

| Subnet | CIDR | VLAN ID |
|--------|------|---------|
| LAN-Users | 192.168.1.0/24 | 10 |
| LAN-Servers | 192.168.10.0/24 | 20 |
| Management | 192.168.100.0/24 | 99 |

### 6.3 Bill of Materials (BOM)

ตารางรายการอุปกรณ์ทั้งหมดที่ต้องจัดซื้อ แสดง:

| คอลัมน์ | ความหมาย |
|---------|----------|
| Category | ประเภทอุปกรณ์ (Router, Switch, Firewall, etc.) |
| Brand / Model | ยี่ห้อและรุ่นที่แนะนำ |
| Specs | Specification สำคัญ เช่น port count, throughput |
| Qty | จำนวนที่ต้องการ |
| Unit Price (THB) | ราคาต่อชิ้น (อ้างอิง ไม่รวม VAT) |
| Subtotal (THB) | ราคารวมต่อรายการ |

ด้านล่างตารางแสดง:
- **รวมทั้งหมด (ก่อน VAT)**
- **VAT 7%**
- **รวมทั้งหมด (รวม VAT)**

> **หมายเหตุสำคัญ:** ราคาอุปกรณ์เป็นราคาอ้างอิงสำหรับการประมาณงบประมาณเบื้องต้นเท่านั้น ราคาจริงอาจแตกต่างตาม vendor, ช่วงเวลา, และเงื่อนไขการจัดซื้อ

### 6.4 Debug Sections (สำหรับผู้ที่สนใจ)

ที่ด้านล่างมี 2 ส่วนที่ซ่อนอยู่ คลิก **"🔍 Parsed Requirements (JSON)"** หรือ **"🔍 Raw Topology JSON"** เพื่อดูข้อมูลดิบที่ AI ประมวลผล ใช้สำหรับตรวจสอบว่า AI เข้าใจ Requirement ถูกต้องหรือไม่

### 6.5 การออกแบบใหม่

กดลิงก์ **"← ออกแบบใหม่"** ที่มุมบนซ้าย เพื่อกลับไปหน้าหลักและกรอก Requirement ใหม่

## 7. ตัวอย่างการใช้งาน

### กรณีที่ 1: ออกแบบเครือข่ายออฟฟิศขนาดกลาง

**Input:**
```
ออฟฟิศ 3 ชั้น ชั้นละ 30 คน มี Server Room 1 ห้อง ต้องการ WiFi ทุกชั้น
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology Type: Tree
- อุปกรณ์: Router 1 ตัว, Core Switch 1 ตัว, Access Switch 3 ตัว, AP 3 ตัว, File Server 1 ตัว
- ราคาประมาณ: 200,000–350,000 บาท (ขึ้นอยู่กับ AI ออกแบบ)

### กรณีที่ 2: เครือข่ายขนาดเล็กที่บ้าน / สำนักงานเล็ก

**Input:**
```
Small home office, 5 people, just need internet and WiFi, very low budget
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology Type: Star
- อุปกรณ์: SMB Router 1 ตัว (พร้อม WiFi ในตัว)
- ราคาประมาณ: 3,000–5,000 บาท

### กรณีที่ 3: เครือข่ายที่ต้องการความปลอดภัยสูง

**Input:**
```
ธนาคารสาขา พนักงาน 20 คน ต้องการ Firewall, redundant internet 2 เส้น
security สูงสุด ห้าม downtime
```

**ผลลัพธ์ที่คาดหวัง:**
- Topology Type: Hybrid
- อุปกรณ์: มี Firewall, Redundant Router, Redundant Switch, UPS
- ราคาประมาณ: 250,000–400,000 บาท

## 8. คำถามที่พบบ่อย

**Q: พิมพ์ภาษาไทยได้ไหม?**  
A: ได้เลย ระบบรองรับทั้งภาษาไทยและอังกฤษ หรือผสมกันก็ได้

**Q: ต้องพิมพ์รายละเอียดมากแค่ไหน?**  
A: อย่างน้อยระบุประเภทองค์กรและจำนวนผู้ใช้ ระบบจะเติม default ที่เหมาะสมให้ส่วนที่เหลือ

**Q: รอนานแค่ไหน?**  
A: ปกติ 10–30 วินาที ขึ้นอยู่กับความซับซ้อนของ requirement และความเร็ว internet

**Q: แผนภาพดาวน์โหลดได้ไหม?**  
A: ปัจจุบัน (MVP) ยังไม่มีปุ่ม Export โดยตรง สามารถคลิกขวาที่รูปแล้ว "Save image as..." ได้

**Q: ราคาในตาราง BOM เชื่อถือได้แค่ไหน?**  
A: เป็นราคาอ้างอิงสำหรับประมาณงบประมาณเบื้องต้นเท่านั้น ควรติดต่อ vendor เพื่อขอ quote จริง

**Q: ถ้า AI ออกแบบผิด Topology ทำอย่างไร?**  
A: กลับไปหน้าหลัก เพิ่มรายละเอียดใน Requirement ให้ชัดเจนขึ้น แล้ว Generate ใหม่

**Q: ระบบเก็บข้อมูล Requirement ของเราไหม?**  
A: ไม่ ระบบ MVP นี้ไม่มี database ข้อมูลไม่ถูกบันทึก
